# 假新闻判断器 — 产品需求文档

**版本**: v0.1  
**日期**: 2026-05-25  
**状态**: 草稿

---

## 一、产品概述

用户上传一张新闻截图，系统自动识别图中的新闻来源（媒体 logo、公众号名、网站、App 名等），并结合 AI 判断该新闻的可信度，输出真假评估结论和依据。

---

## 二、核心使用场景

| 场景 | 描述 |
|------|------|
| 朋友圈/群消息 | 收到一条"震惊体"截图，想快速鉴别真假 |
| 微博/微信公众号 | 截图某条新闻，验证媒体来源是否可信 |
| 境外新闻 | 识别外媒 logo，判断是否断章取义或伪造 |

---

## 三、功能需求

### 3.1 图片上传与验证（前置门槛）

**图片必须包含以下至少一项，否则拒绝处理**：

- 媒体/机构 **logo**（如央视、人民日报、BBC、CNN 等）
- **公众号名称** / **账号名**（如"@xxx新闻"）
- **网站域名**（如 xinhua.net、thepaper.cn）
- **App 名称**（如"澎湃新闻"、"今日头条"）
- **节目/栏目名称**（如"新闻联播"水印）

> **拒绝场景举例**：纯文字截图、无来源标识的图片、只有正文没有媒体标识的截图  
> **系统提示语**："图片中未检测到可识别的新闻来源，请上传含媒体标识的截图"

### 3.2 新闻来源识别

AI 从图片中提取：

- 媒体名称（中英文）
- 媒体类型（官方媒体 / 自媒体 / 境外媒体 / 未知）
- 识别置信度（高 / 中 / 低）

### 3.3 可信度分析

基于以下维度输出结论：

| 分析维度 | 说明 |
|----------|------|
| 来源可信度 | 该媒体的历史可信记录、是否为已知造谣来源 |
| 标题/内容特征 | 是否含"震惊"、"曝光"、"独家"等低质标题词 |
| 内容逻辑 | 数字、时间、地点是否自洽，是否存在明显矛盾 |
| 视觉篡改线索 | 图片是否有明显拼接、字体异常、水印位置错误等 |
| 已知事实核查 | 结合 AI 知识库，判断内容是否与已知事实严重矛盾 |

**输出结论等级**：

- ✅ 可信 — 来源权威，内容无明显问题
- ⚠️ 存疑 — 来源或内容存在可疑点，建议核实
- ❌ 疑似虚假 — 发现明确造假信号或严重事实错误
- ❓ 无法判断 — 信息不足，无法给出结论

### 3.4 输出报告

每次分析返回：

```
【来源】澎湃新闻（官方媒体，可信度：高）
【结论】⚠️ 存疑
【理由】
  1. 标题使用"震惊"等夸大用词
  2. 文中数据与已知公开数据存在出入
  3. 未检测到明显视觉篡改
【建议】建议在澎湃新闻官网搜索原文核实
```

## 四、技术方案

### 4.1 AI 后端（免费方案选型）

推荐使用 **Google Gemini Flash**（免费额度最充裕且支持视觉）：

| 方案 | 视觉能力 | 免费额度 | 推荐度 |
|------|----------|----------|--------|
| Gemini 1.5 Flash | ✅ 强 | 1500次/天 | ⭐⭐⭐ 首选 |
| Groq + LLaMA Vision | ✅ 有限 | 有速率限制 | ⭐⭐ 备选 |
| OpenAI GPT-4o-mini | ✅ 强 | 极少免费额度 | ⭐ 不推荐免费用 |

> 申请地址：Google AI Studio → 获取 API Key（免费，无需信用卡）

### 4.2 技术栈（建议）

- **前端**：单页 Web App，HTML + TailwindCSS（无框架，简单快速）
- **后端**：Python Flask 或 Node.js（单文件即可）
- **部署**：本地运行 / Vercel / Railway（均有免费层）

### 4.3 核心调用流程

```
用户上传图片
    ↓
前端预检（文件大小 < 10MB，格式：jpg/png/webp）
    ↓
发送到后端
    ↓
调用 Gemini Vision API（传图 + Prompt）
    ↓
解析 AI 返回的结构化 JSON
    ↓
渲染结果页面
```

### 4.4 Prompt 设计要点

发给 AI 的 Prompt 需包含：

1. 角色设定：专业新闻核查员
2. 任务一：检查图中是否有媒体来源标识（无则拒绝）
3. 任务二：识别媒体名称和类型
4. 任务三：分析内容可信度
5. 输出格式：要求返回 JSON，方便解析
